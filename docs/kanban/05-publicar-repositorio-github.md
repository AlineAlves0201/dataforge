# 05 - Publicar repositorio no GitHub

## Objetivo

Enviar o projeto DataForge para o GitHub.

## Entrega

Repositorio remoto `dataforge` publicado e sincronizado.

## Checklist

- Rodar `gh auth login`.
- Criar o repositorio com `gh repo create dataforge --private --source=. --remote=origin --push`.
- Conferir o remoto com `git remote -v`.
- Abrir o repositorio no navegador.

## Criterio de aceite

O projeto aparece no GitHub com os arquivos da Sprint 1.
